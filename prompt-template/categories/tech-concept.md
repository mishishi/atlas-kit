# 分类模板 · 科技概念

> 用法：接在主模板之后。用本模板的内容替换主模板中的主视觉逻辑、8 个模块栏目、评分维度，并叠加防翻车规则。

## 强调色

青蓝 `#2A8FA8`

## 固定 8 模块

1. 概念档案
2. 核心定义
3. 基本原理
4. 关键参数
5. 应用场景
6. 发展历程
7. 局限挑战
8. 快速评分卡

## 评分维度

技术成熟度 / 应用广泛度 / 颠覆潜力 / 普通人可及度 / 争议程度

## 观察区逻辑

科技概念多为抽象事物，往往没有「实物主视觉」。主视觉用「核心原理示意图 / 结构框图」担当，而非具象照片。

观察区放 3–5 个：原理关键环节 / 核心组件或模块 / 输入输出示意 / 典型应用图标 / 数据流或信号流。用「原理」「模块」「应用」等短标题，不编号。抽象概念允许用图标化、几何化表达。

## 可视化条带

- 简化结构图或原理示意图（流程图 / 模块图风格）
- 输入输出流程图：展示数据、处理模块、结果之间的关系
- 应用场景图标带：用 3–5 个图标标出典型使用场景

## 一句话锚定

用户第一眼必须认为：这是一本科技百科里的概念页，用结构图和数据拆解这项技术如何运作。

而不是：科技公司产品发布海报、炫酷概念渲染图、赛博朋克风插画。

## 防翻车规则

- 不做成产品发布会海报、未来感概念大片、赛博朋克霓虹风
- 不出现「颠覆」「革命性」「黑科技」等营销夸张词
- 原理示意求清晰准确，不堆砌看着酷但无意义的科幻装饰
- 参数用量级或范围，不编造精确 benchmark 数字
- 发展历程用确定里程碑，不预测性地虚构未来时间点
- 应用场景如实陈述，不夸大尚未实现的能力
